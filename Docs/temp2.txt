## 2. Celková architektura
### 2.1. Přehled architektury
Utilita bude navržena jako JavaScriptový modul (dle NFR-002, implementace jako RequireJS modul), který bude poskytovat sadu funkcí pro Widgety. Veškerá komunikace s platformou 3DEXPERIENCE bude zapouzdřena a bude probíhat výhradně přes existující modul `Connector3DSpace.js` (dle NFR-001).

Utilita bude interně implementovat vícekrokové procesy vyžadované Document Web Services, jako je získávání ticketů pro FCS, nahrávání/stahování souborů na/z FCS a následné potvrzování operací vůči Document objektům.

**Základní tok interakce komponent:**
```plantuml
@startuml
actor Widget
participant "DYTUtils (Doc Extension)" as Utilita
participant "Connector3DSpace.js" as Connector
database "3DEXPERIENCE\n(Document WS, FCS)" as Platforma

Widget --> Utilita : Volání API funkce
Utilita --> Connector : Volání pro 3DX operaci
Connector --> Platforma : HTTP požadavek
Platforma --> Connector : HTTP odpověď
Connector --> Utilita : Výsledek operace
Utilita --> Widget : Výsledek / Chyba
@enduml
